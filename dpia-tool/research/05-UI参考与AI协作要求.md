# UI 参考与 AI 协作要求

## 1. AI coworker 要求的当前状态

用户新增要求之一，是在需求层面引入 `AI coworker` 讨论，形成第二视角。

本轮已做的可验证检查包括：

- 搜索工作区内是否存在现成 coworker 集成
- 检查环境变量中是否存在常见模型 API 密钥
- 检查本机是否存在可直接调用的本地模型命令
- 检查 MCP 资源与模板中是否存在可用的第二模型入口

当前结论：

- 没有发现可直接调用的第二 AI 模型入口
- 没有发现已配置好的 coworker 工具或凭证
- 因此，本轮不能诚实地声称“已经与独立 AI coworker 完成讨论”

这个要求应保留为正式交付前的必经环节。

### 1.1 对产品流程的要求

后续在原型、信息架构或实现阶段，应增加一条正式工作要求：

- 关键交付物必须经过第二 AI 模型或独立 AI coworker 的 review

review 输出至少应包括：

- 同意点
- 不同意点
- 风险提醒
- 建议修改项

## 2. LIA tool 与 Data Compliance Tool 的 UI 共性

这两套现有产品已经形成了稳定的工作台设计语言，应复用的不只是风格，还包括版式节奏。

应继承的共性包括：

- 浅色、带柔和渐变背景
- 玻璃质感白色面板
- 大圆角卡片与胶囊式导航
- 宽屏工作台布局
- 标题层次非常明确
- 模块与模块之间的节奏统一
- 输入卡片和操作按钮之间留白充分

## 3. 必须复用的版式与间距基线

以下数值来自现有两个工具的 CSS，应视为实现基准：

### 3.1 页面外层

- 内容宽度：`min(1480px, calc(100% - 40px))`
- 页面顶部/底部内边距：`18px 0 56px`
- 主内容区模块间距：`24px`

### 3.2 顶部导航

- 吸顶距离：`top: 16px`
- 底部外边距：`28px`
- 内边距：`10px 14px` 到 `12px 14px`
- 圆角：`999px`
- 内部间距：`14px` 到 `16px`

### 3.3 Panel 卡片

- 主卡片圆角：`30px`
- 次级卡片圆角：`24px`
- 小卡片圆角：`18px`
- 使用半透明白底 + `blur(18px)`
- 主模块常见内边距：`24px`
- 大型工作台模块内边距：`28px`

### 3.4 Section heading

- 内部主轴间距：`18px` 到 `24px`
- label 与标题之间：`8px`
- 标题区与下方主体内容间距：`16px` 到 `22px`

### 3.5 文本层级

- 小标签 `section-label`：
  - 字号约 `0.76rem` 到 `0.8rem`
  - 全大写
  - 字距 `0.12em` 到 `0.18em`
- 一级标题 `h1`：
  - 行高约 `0.92` 到 `1.04`
- 二级标题 `h2`：
  - 行高约 `0.98` 到 `1.16`
- 正文说明：
  - 行高约 `1.58` 到 `1.7`

### 3.6 Grid 与表单卡片

- `field-grid` 间距：`14px` 到 `16px`
- 字段卡片内边距：`16px` 到 `18px`
- 字段卡片头部与正文距离：约 `12px`

### 3.7 Chip / Badge / Button

- chip / badge 最小高度：`36px` 到 `38px`
- button 最小高度：`42px` 到 `44px`
- button 横向内边距：`16px`

## 4. 对 DPIA 工具 UI 的明确要求

后续 DPIA tool 在 UI 上必须满足：

- 风格对齐 `lia-tool` 和 `data-compliance-tool`
- 同时复用标题、正文、模块和表单之间的垂直节奏
- 所有主模块默认采用 `24px` 级别内边距
- 相邻工作区模块默认采用 `24px` 级别间距
- 顶部导航保留胶囊形吸顶结构
- 标签、标题、正文的字号和行高区间不得随意漂移

## 5. 不可接受的实现方式

以下实现应视为不合格：

- 只复用颜色，不复用布局比例
- 只做类似卡片，不继承标题和正文节奏
- 标题、正文、字段卡片、模块之间间距不一致
- 新页面看起来像另一套设计系统
